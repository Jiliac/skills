# Product Brief Quality Checklist

Run this checklist before finalizing the brief. Flag any issues to the user.

## Completeness

- [ ] Executive summary clearly communicates vision and problem
- [ ] Problem statement is specific and quantifiable
- [ ] Target users are well-defined with compelling personas
- [ ] Success metrics connect user value to business objectives
- [ ] MVP scope is focused and realistic
- [ ] Future vision is present and inspiring

## Coherence

- [ ] All sections align with the core problem statement
- [ ] User value is consistently emphasized throughout
- [ ] Success criteria are traceable to user needs and business goals
- [ ] MVP scope aligns with the problem, solution, and user needs
- [ ] Personas are referenced in success metrics and scope sections

## Quality

- [ ] No vague metrics — everything measurable or observable
- [ ] No scope creep — MVP is genuinely minimal and viable
- [ ] Differentiators are real advantages, not aspirational claims
- [ ] User personas feel like real people, not demographic summaries
- [ ] Out-of-scope items have clear rationale for deferral

## Actionability

- [ ] Brief provides clear direction for PRD creation
- [ ] Success metrics can drive acceptance criteria
- [ ] MVP scope is implementable, not just a wish list
- [ ] Next steps are identified
